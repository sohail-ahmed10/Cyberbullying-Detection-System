# 🛡️ Cyberbullying Detection System

An advanced Applied Machine Learning Evaluation and Auditing Platform designed to detect and classify various forms of cyberbullying in text patterns (Tweets/Comments) using Natural Language Processing (NLP) and robust Classification Pipelines.

---

## 🚀 Key Features
* **Multi-Model Benchmark Integration:** Features a comprehensive model evaluation engine built on Logistic Regression, Random Forest, and Support Vector Machines (SVM).
* **Industrial Preprocessing Pipeline:** Uses NLTK for structured Stop-Words filtering, token regularizations, and WordNet Lemmatization.
* **Modern Glassmorphic UI Dashboard:** Premium frosted-glass responsive interface tracking real-time analytics indicators (Total Analyzed, Threats Detected, Safe Content).
* **Live Cloud Synchronization:** Real-time database tracking logs directly synchronized with MongoDB Atlas Cloud infrastructure using secure SSL handshake rules.

---

## 🛠️ Tech Stack Architecture
* **Frontend Layer:** Semantic HTML5, Dynamic CSS3 (Frosted Glass Custom Modules), JavaScript (Asynchronous Fetch API Matrix)
* **Backend Framework:** Python Flask, Flask-CORS (Cross-Origin Resource Sharing Engine)
* **Machine Learning & Data Science:** Scikit-Learn, Pandas, NumPy, NLTK, Jupyter Notebook
* **Database Management:** MongoDB Atlas (Cloud Database Service Engine)

## ⚙️ Installation & Deployment Guidelines
Follow these exact operational steps to configure and run the application platform locally:
### 1. Environment Setup & Dependencies Installation
Navigate to your backend deployment directory terminal and execute:
bash
cd backend
pip install -r requirements.txt


### 2. Executing the Machine Learning Workflow (Optional)
To review dataset structures or re-evaluate pipelines using raw data sources:
bash
python train_model.py


### 3. Deploying the Live Local Server Application
Execute the primary execution file to initiate the network listener:
bash
python app.py


Once the server initialization process prints the structural database confirmations, navigate to your host routing configuration via any browser interface:
text
[http://127.0.0.1:5000](http://127.0.0.1:5000)


## 🔒 Security & Privacy Features
 * Handshake requests initiated between the server modules and the cloud database clusters are processed through validated SSL environments managed dynamically via Python's local certifi architecture.
 * Real-time text submissions are securely processed through transactional memory blocks and logged in the database cluster with encrypted UTC timestamps.
